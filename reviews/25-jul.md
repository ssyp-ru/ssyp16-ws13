Не надо так делать.
И вообще не надо модули использовать.


fsFunctions.ts
--------------

Идея функций здесь заключается в том, чтобы из вот этих 
кидающих исключение функций сделать функции, которые 
заворачивают `try/catch` внутри себя, и при случае 
исключения возвращают `null` (если это имеет смысл как у 
функции `statSync`).

К примеру, так:

```statSync(p) { try { return fs.lstatSync(p); } catch (e) { return null; }}```

Ну и убрать из них слово Sync.
